## 新增需求

### 需求：生成设计概念故事
系统须生成引人入胜的设计概念故事（300-500 中文字），将主题、品牌定位与空间体验串联起来。

#### 场景：概念故事生成
- **当** 需求分析 Agent 提供统一需求，且知识检索 Agent 提供案例摘要
- **则** 概念设计 Agent 以"资深美陈设计师"人设调用 LLM
- **且** 生成包含以下要素的故事：叙事钩子、空间体验旅程、情感共鸣点、与目标人群的连接
- **且** 故事最多引用 2 个相似历史案例以增强可信度

#### 场景：用户驳回概念故事
- **当** 用户反馈故事"太幼稚"或要求调整方向
- **则** 系统按修改后的调性关键词（如"更高端"、"更年轻"）重新生成故事
- **且** 保留核心主题，调整情感定位

### 需求：生成氛围描述
系统须为设计概念生成 8-10 个氛围关键词及一段约 200 字的感官描述。

#### 场景：氛围生成
- **当** 概念故事已获确认
- **则** 系统生成分类氛围关键词：视觉类（3个）、触觉类（2个）、听觉类（2个）、嗅觉类（1个，可选）
- **且** 生成一段感官段落，描述访客进入空间后的体验

### 需求：生成 Moodboard
系统须生成包含 6-9 张参考图的 Moodboard，以可视化概念方向。

#### 场景：Moodboard 生成
- **当** 概念故事和氛围描述已获确认
- **则** 系统基于概念生成 2-3 个图像生成 Prompt
- **且** 调用图像生成服务生成概念 Moodboard 图
- **且** 从 Milvus 向量数据库检索 3-6 张额外历史参考图（如有）补充 AI 生成图
- **且** 将所有图片以网格布局排列，标注 image_type（AI生成或历史参考）及 relevance_note
