## 新增需求

### 需求：合并多源输入为统一需求
系统须将输入解析器（照片、CAD、参考图、文本）的输出合并为单一的结构化设计需求文档。

#### 场景：提供全部输入
- **当** 输入解析器已完成现场照片、CAD、参考图和文本需求的处理
- **则** 需求分析 Agent 生成符合 DesignRequirement 结构的统一需求 JSON
- **且** 冲突信息（如文本写预算15万但照片暗示高端商场）被标记，等待用户确认

#### 场景：仅提供部分输入
- **当** 仅提供文本需求和参考图（无 CAD 或照片）
- **则** 系统用已有字段生成需求 JSON
- **且** 缺失的空间信息标记为"待确认"
- **且** 建议用户上传现场照片以提高准确性

### 需求：预算档次分类
系统须根据空间类型和预算金额，将项目预算划分为低、中、高三档。

#### 场景：预算分类
- **当** 需求 JSON 包含 budget_amount 和 space_type
- **则** 系统查询 SQLite 历史数据库中的同类空间数据
- **且** 基于历史数据分位数将预算划分为低/中/高
- **且** 输出中包含分类依据说明

### 需求：提取主题与风格方向
系统须从用户输入和参考图中推导具体的主题方向和风格关键词。

#### 场景：基于参考图提取主题
- **当** 用户提供主题关键词"夏日海洋"及参考图
- **则** 系统综合生成详细风格方向，包括：核心视觉隐喻、色板（hex）、材料建议、灯光氛围
- **且** 风格方向与文本输入及参考图分析保持一致

### 需求：识别约束与风险
系统须从用户输入中识别显式和隐式约束，包括材料限制、时间约束及法规影响。

#### 场景：约束识别
- **当** 用户提到"工期20天"和"不能用真植物"
- **则** 系统提取：hard_deadline=20_days、material_blacklist=["real_plants"]
- **且** 生成隐性风险提示："20天工期可能限制定制加工，建议考虑模块化/租赁方案"
