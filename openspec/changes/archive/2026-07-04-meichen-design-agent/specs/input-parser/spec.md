## 新增需求

### 需求：解析现场照片与视频
系统须分析用户上传的现场照片和视频，提取空间信息，包括层高、面积、现有装饰、柱位和人流动线。

#### 场景：上传单张或多张现场照片
- **当** 用户上传一张或多张现场照片
- **则** 系统调用 VLM（Qwen2.5-VL）生成空间结构化描述
- **且** 描述包含：空间类型、预估尺寸、现有视觉元素、光照条件、照片中可见的约束

#### 场景：上传视频
- **当** 用户上传现场漫游视频
- **则** 系统按 2 秒间隔抽取关键帧
- **且** 将关键帧作为批量照片处理，聚合空间描述

### 需求：解析 CAD 图纸
系统须解析上传的 CAD 文件（DWG/DXF），提取平面轮廓、柱网位置和面积尺寸。

#### 场景：上传有效的 CAD 文件
- **当** 用户上传 DWG 或 DXF 文件
- **则** 系统使用 ezdxf 解析文件
- **且** 提取：外轮廓多边形、柱中心点、总面积、包围盒尺寸
- **且** 渲染一张带透明背景的平面图 PNG 预览

#### 场景：CAD 解析失败
- **当** 用户上传 ezdxf 无法解析的 CAD 文件
- **则** 系统返回解析失败原因的错误信息
- **且** 提示用户上传现场照片或 PDF 平面图作为降级方案

### 需求：解析参考图
系统须分析参考图（Pinterest、小红书、竞品案例），提取视觉风格属性，包括色板、材质偏好、造型语言和空间氛围。

#### 场景：上传参考图
- **当** 用户上传一张或多张参考图
- **则** 系统调用 VLM 为每张图生成风格分析
- **且** 分析包含：主色调、材质类型、装饰元素、灯光风格、整体氛围关键词
- **且** 系统基于视觉 Embedding 相似度对参考图聚类并去重

### 需求：解析 PDF 文件
系统须解析用户上传的 PDF 需求书或参考文档，提取文本内容和内嵌图片。

#### 场景：上传 PDF 需求书
- **当** 用户上传 PDF 文件
- **则** 系统使用 pdfplumber 或 PyMuPDF 提取全部文本内容
- **且** 使用 PyMuPDF 提取 PDF 中的内嵌图片
- **且** 将提取的文本送入文本需求提取器，将提取的图片作为参考图处理
- **且** 若 PDF 为扫描版（无文本层），使用 VLM 对每页进行 OCR 分析

#### 场景：PDF 解析失败
- **当** PDF 文件加密、损坏或格式异常导致解析失败
- **则** 系统返回错误信息，标记该文件为"未解析"
- **且** 不阻断其他输入的解析流程

### 需求：解析 PPT 文件
系统须解析用户上传的 PPT 参考方案或历史项目文件，提取幻灯片文本和缩略图。

#### 场景：上传 PPT 参考文件
- **当** 用户上传 PPT 文件（.pptx）
- **则** 系统使用 python-pptx 提取每张幻灯片的文本内容
- **且** 将幻灯片渲染为 PNG 缩略图（或使用 python-pptx 提取内嵌图片）
- **且** 将提取的文本作为辅助需求材料，将幻灯片图片作为参考图送入 VLM 分析
- **且** 返回每张幻灯片的结构：slide_index、text_content、image_count

#### 场景：PPT 解析失败
- **当** PPT 文件为旧格式（.ppt）或损坏导致解析失败
- **则** 系统返回错误信息，建议用户转换为 PDF 后重新上传
- **且** 不阻断其他输入的解析流程

### 需求：解析文本需求
系统须从用户提供的自然语言输入中提取结构化设计约束。

#### 场景：自然语言需求输入
- **当** 用户提供如"夏日海洋主题中庭吊饰，预算15万，工期20天，不能用真植物"等文本
- **则** 系统调用 LLM 提取 JSON 对象，包含：theme_keywords、budget_amount、budget_level、space_type、deadline_days、material_restrictions、target_audience、special_notes
- **且** 缺失字段明确标记为 null，而非臆造
