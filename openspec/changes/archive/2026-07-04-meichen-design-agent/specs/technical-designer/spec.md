## 新增需求

### 需求：生成带安装点位的平面图
系统须生成一张标注图，在 CAD 推导的平面图上显示建议的安装位置、物品编号及大致尺寸。

#### 场景：基于 CAD 标注平面图
- **当** CAD 解析成功且 L2 视觉方案已确认
- **则** 可落地设计 Agent 分析空间布局，基于人流动线、视觉焦点、结构约束提出 3-8 个安装点位
- **且** 使用 Pillow 生成标注 PNG：CAD 轮廓作为底图 + 编号圆圈标注点位 + 尺寸线 + 图例
- **且** 每个点位包含：item_id、item_name、approximate_size、height_from_floor

#### 场景：仅基于照片标注平面图
- **当** 无 CAD 但上传了现场照片
- **则** 系统基于照片分析生成简化示意布局
- **且** 大致放置安装点位，并附注："点位示意，需现场复核"

### 需求：生成物料规格清单
系统须生成详细的物料规格清单（物料清单），包含制作和安装所需的全部物品。

#### 场景：物料清单生成
- **当** L2 视觉方案和平面图已确认
- **则** 系统将设计拆解为单品：结构框架、装饰元素、灯光、画面、安装五金
- **且** 生成 JSON 数组，每项包含：item_id、item_name、specification、material、finish、quantity、unit、supplier_suggestion、reference_image_url
- **且** 数量估算参考平面图面积和知识库中的典型密度参数

#### 场景：材料限制合规
- **当** 需求中包含材料限制（如"不能用真植物"）
- **则** 系统在清单生成前过滤掉受限材料
- **且** 自动从知识库推荐替代方案（如"仿真植物"或"纸艺花卉"）

### 需求：生成预算汇总
系统须生成一份带分项成本和合计的预算汇总表。

#### 场景：预算计算
- **当** 物料规格清单已完成
- **则** 系统查询 material_price 数据库的单价
- **且** 计算：材料费、制作费（按复杂度估算为材料费的 30-50%）、安装费（按点位数估算）、设计费
- **且** 生成表格，包含：category、item_count、subtotal、notes
- **且** 合计与用户预算对比，超支 10% 以上时标红警告

#### 场景：缺失价格数据
- **当** 某种材料在数据库中无价格
- **则** 系统将该价格标记为"参考价待确认"
- **且** 附注："建议询价供应商"
- **且** 该物品仍计入合计，但以虚线表示估算值
