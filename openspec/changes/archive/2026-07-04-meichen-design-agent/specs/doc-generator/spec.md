## 新增需求

### 需求：生成 HTML 预览
系统须生成一份响应式 HTML 文档，以可导航的章节预览完整设计方案。

#### 场景：生成 L1 HTML
- **当** 用户请求 L1 概念的 HTML 输出
- **则** 文档生成器渲染 Jinja2 模板，包含：封面（项目名称、日期）、概念故事、氛围关键词、Moodboard 图片网格
- **且** HTML 为自包含文件，图片使用 Base64 内嵌，CSS 内联
- **且** HTML 可在 Chrome/Safari 中正确打开，无需外部依赖

#### 场景：生成 L3 HTML
- **当** 用户请求完整 L3 交付物的 HTML 输出
- **则** 系统生成多章节 HTML，包含：封面、概念、效果图、色彩材质板、平面图、物料清单（可排序表格）、预算汇总
- **且** 包含适合打印的 CSS 媒体查询

### 需求：生成 PPT 汇报文档
系统须生成适用于甲方汇报的 PowerPoint 演示文稿（.pptx）。

#### 场景：基于模板生成 PPT
- **当** 用户请求 PPT 输出
- **则** 文档生成器加载母版 PPT 文件（template.pptx），使用预定义版式：标题页、概念页、图文页、表格页、全图页
- **且** 将方案内容映射到对应版式：概念故事 → 图文页，效果图 → 全图页，物料清单 → 表格页
- **且** 将占位图替换为实际生成图片，并调整至幻灯片尺寸
- **且** 输出标准 16:9 比例的 .pptx 文件

#### 场景：模板缺失
- **当** 项目目录中未找到 template.pptx
- **则** 系统使用 python-pptx 程序化生成最小默认模板
- **且** 记录警告："使用默认模板，如需品牌输出请自定义 template.pptx"

### 需求：生成 PDF 文档
系统须生成适合归档和打印的 PDF 文档。

#### 场景：PDF 生成
- **当** 用户请求 PDF 输出
- **则** 系统先生成 HTML 预览
- **且** 使用 WeasyPrint 或 Playwright 无头浏览器将 HTML 转为 PDF
- **且** 应用 A4 页边距和页码
- **且** 确保图片分辨率不低于 150 DPI，满足打印质量

#### 场景：PDF 生成失败
- **当** WeasyPrint/Playwright 失败（如缺少系统依赖）
- **则** 系统降级为通过 LibreOffice 无头模式从 PPT 文件转换 PDF
- **且** 提示用户："PDF 由 PPT 转换生成，排版可能与 HTML 预览略有差异"
